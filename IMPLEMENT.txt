## Store State (appStore.ts)

```typescript
interface Settings {
  volumeTriggerEnabled: boolean;    // Volume button trigger
  shakeTriggerEnabled: boolean;     // Shake trigger
  smsEnabled: boolean;             // SMS alerts
  callEnabled: boolean;             // Auto phone call
  flashEnabled: boolean;            // Flashlight SOS
  vibrationEnabled: boolean;        // Vibration
  voiceTriggerEnabled: boolean;     // Pro: Voice trigger
  fallDetectionEnabled: boolean;   // Pro: Fall detection
  autoRecordEnabled: boolean;      // Pro: Auto video recording
  useFrontCamera: boolean;        // Pro: Camera preference
  testMode: boolean;              // Test mode master switch
  testAlarm: boolean;             // Test: disable alarm
  testSMS: boolean;               // Test: disable SMS
  testCalls: boolean;             // Test: disable calls
}

interface AppState {
  emergencyContacts: EmergencyContact[];
  isAlarmActive: boolean;
  settings: Settings;
  lastAlertTime: Date | null;
  lastLocation: string | null;
  alertHistory: AlertRecord[];
  isPro: boolean;
  subscriptionExpiry: Date | null;
}
```

## Test Mode Logic

When Test Mode is enabled, each toggle controls what is DISABLED:

| Toggle | When ON | Effect |
|--------|---------|--------|
| Enable Test Mode | Master switch | Activates test mode |
| Alarm Sound | Disables | Alarm won't play |
| SMS Alerts | Disables | SMS won't send |
| Auto Phone Call | Disables | Call won't happen |

2. Volume button detection needs native code implementation
3. Flash service uses placeholder (CameraView required for torch)
4. Voice trigger needs native voice recognition setup
5. Fall detection uses accelerometer algorithm
6. All Pro features have UI but need RevenueCat setup for real purchases
